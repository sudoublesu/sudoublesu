执行摘要 Executive Summary

本项目旨在构建一套先进的AI辅助城市设计系统，通过**AI Agent架构**，实现城市设计前期**现状认知、问题诊断与策略生成**的自动化与智能化。系统将整合多源数据，利用大型语言模型（LLM）进行专业推理，并支持模块化、可扩展的人机协作。首阶段聚焦于五大核心分析维度，旨在显著提升城市设计的效率与科学性。

项目开发结构与阶段划分 Development Phases

阶段一：

现状认知与策略生成

利用现有空间数据与规划资料，对项目地块进行功能、风貌、交通、空间等方面的系统评估，输出具备专业洞察力的图件、文字、策略建议，为后续设计提供智能支撑。本阶段将重点解决数据整合、基础分析、初步策略生成及可视化输出问题。

阶段二：

方案辅助生成与智能优化

在策略框架基础上，结合给定指标和设计原则，实现自动生成多种空间布局草图、三维体量模型，并提供性能评估与设计改进建议。此阶段将引入更复杂的生成式AI和优化算法。

项目概况识别模块 Project Overview

工作流程

  • 梳理地块区位、交通条件、周边建设、用地现状、土地权属
  • 输出项目基础图件和文字说明

数据依赖

  • 项目边界（Polygon）
  • 土地使用现状（GeoJSON）
  • 地籍图（含权属信息）
  • 道路/轨交/公交点位、公共服务POI点

输出目标

  • 区位图、交通图、用地图、权属图
  • 项目概况文字说明（报告/PPT）

Agent结构

LocationContext Agent, TransportAccess Agent, LanduseBuild Agent, LandOwnership Agent, ContextNarrative Agent

功能分析模块 Function Analysis

工作流程

  • 分析现状功能与产业分布
  • 结合人口/POI/产业趋势识别需求
  • 参考上位规划、案例研究提出功能定位与布局策略

数据依赖

  • 用地现状、企业数据、信令数据、人口数据
  • 公共设施POI、相关规划文本

输出目标

  • 功能结构图、热力图、OD图
  • 案例对比分析、产业建议段落
  • 概念功能布局草图

Agent结构

FunctionSpatial Agent, IndustryTrends Agent, DemandMatch Agent, PlanningInterpret Agent, BenchmarkCases Agent, FunctionStrategy Agent, FunctionLayout Agent

风貌分析模块 Landscape Analysis

工作流程

  • 深度分析建筑高度、界面、天际线、风貌分区
  • 识别视觉资源，生成具有专业判断力的风貌策略

数据依赖

  • 建筑轮廓+高度、路网、POI、街景图（选配）
  • 影像资料、规划控规数据、地标信息

输出目标

  • 高度图、界面连续性图、风貌分区图、天际线图
  • 各区设计引导文字（高度控制、立面风格、材质建议等）

Agent结构

MorphoBase Agent, Morpho3D Agent, VisualAsset Agent, StreetInterface Agent, Skyline Agent, CharacterZone Agent, StyleGuide Agent

交通分析模块 Traffic Analysis

工作流程

  • 分析项目区域对外交通关系、内部路网、公共交通与慢行系统
  • 识别断裂点与出行行为，提出组织优化建议

数据依赖

  • 道路网、轨交/公交点位、项目边界、建筑轮廓
  • 手机信令OD数据、街道等级、功能需求节点

输出目标

  • 道路等级图、等时圈图、OD箭图、公交服务盲区图
  • 出入口分析结果 + 交通策略建议

Agent结构

RoadStructure Agent, AccessEval Agent, InternalNetwork Agent, PedestrianFlow Agent, TransitCoverage Agent, ODPattern Agent, TransportStrategy Agent

空间分析模块 Space Analysis

工作流程

  • 构建空间骨架、评估可达性与街坊尺度
  • 识别断裂与空间潜力，提出空间结构优化策略

数据依赖

  • 建筑轮廓、道路网、绿地广场数据、街角空间、POI
  • 项目边界、开放空间面数据、街景（选配）

输出目标

  • 空间结构图、渗透性图、断裂图、街坊尺度图
  • 潜力点图层 + 策略建议段落

Agent结构

CorePattern Agent, Permeability Agent, OpenSpace Agent, BlockMorphology Agent, Disconnection Agent, SpatialPotential Agent, SpatialStrategy Agent

技术与接口建议 Technology & Interfaces

标准化接口

所有 Agent 均设计为标准化可调用的函数/服务接口（支持 API/Function 调用），确保模块间松耦合。

用户交互

支持用户输入互动与 AI 对话式反馈，并记录用户反馈以进行迭代学习。

多格式输出

图件支持 .png/.svg/GeoJSON，文本支持 Markdown、PDF、Word，具备结构化和版本管理。

本地部署

所有模块需支持本地部署，数据不得上传至公网，确保数据安全和隐私。

推荐技术栈

Python（GeoPandas、NetworkX、LLM接口）、前端 CesiumJS 或 Streamlit。

优先开发Skyline Agent Prioritizing Skyline Agent

核心优势

  • **明确痛点与直观可视化：** 天际线分析成果直接影响城市形象，易于展示。
  • **技术路径清晰：** 数据（建筑高度）易得，计算逻辑成熟，可视化效果直观。
  • **LLM赋能价值高：** 将统计数据转化为专业、有理由的城市设计建议，提升决策科学性。
  • **验证Agent架构：** 作为风貌模块核心，验证独立开发与集成可行性。
  • **承前启后作用：** 为后续模块提供基础高度数据。
  • **早期演示效果显著：** 快速展示系统潜力与价值。

开发深化思路

  • **基础功能实现：** 遵循现有Python脚本逻辑。
  • **参数可配置化：** 允许用户调整缓冲区、高度比例系数。
  • **LLM Prompt优化：** 持续迭代Prompt，加入更多城市设计上下文。
  • **结果可视化优化：** 颜色渐变、过高/过低建筑突出显示。
  • **接口标准化：** 确保输入输出符合统一API规范。

未来展望

可扩展至多种天际线视点分析、高度剖面图生成，提供更精细的城市设计辅助。
